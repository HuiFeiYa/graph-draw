# HfDraw 图形渲染引擎架构文档

## 项目概述

`@hfdraw/graph` 是 HfDraw 项目的核心图形渲染引擎包，负责处理所有图形元素的渲染、交互和管理。该包基于 Vue 3 和 SVG 技术构建，提供了完整的图形绘制和编辑功能。

## 技术栈

- **Vue 3**: 组件化框架，提供响应式渲染
- **TypeScript**: 类型安全的开发语言
- **SVG**: 矢量图形渲染技术
- **Vite**: 构建工具（开发环境）
- **@hfdraw/types**: 类型定义包
- **@hfdraw/utils**: 工具函数包
- **@hfdraw/elbow**: 路径和曲线绘制工具包

## 核心架构

### 1. 主要组件架构

#### GraphView.vue - 主视图组件
- **功能**: 图形画布的主容器，处理用户交互
- **特性**:
  - 支持拖拽、缩放、平移操作
  - 管理视图变换和坐标系统
  - 集成各种子组件（图形、选择框、标签编辑器等）
  - 处理鼠标和键盘事件

#### DiagramShape.vue - 图形渲染组件
- **功能**: 根据图形类型动态渲染不同的图形组件
- **特性**:
  - 使用 `shapeCompManager` 管理图形组件映射
  - 支持符号图形和边线图形的渲染
  - 提供统一的图形渲染接口

### 2. 数据模型架构

#### GraphModel.ts - 核心图形模型
- **功能**: 管理整个图形的数据状态
- **包含模型**:
  - `ViewModel`: 视图状态（缩放、平移、变换）
  - `MoveModel`: 移动操作状态
  - `SelectionModel`: 选择状态管理
  - `HoverModel`: 悬停状态管理
  - `LabelEditorModel`: 标签编辑状态
- **数据管理**:
  - `shapes`: 所有图形元素集合
  - `edges`: 边线元素集合
  - 响应式数据更新和状态同步

#### 各子模型职责
- **ViewModel**: 管理画布的视图变换、缩放比例、平移偏移
- **MoveModel**: 处理图形移动操作的开始、进行、结束状态
- **SelectionModel**: 管理图形选择状态，支持单选和多选
- **HoverModel**: 处理鼠标悬停效果和高亮显示
- **LabelEditorModel**: 管理文本标签的编辑状态和位置

### 3. 图形组件系统

#### 图形类型支持
- **SymbolShape**: 基础符号图形（矩形、圆形等）
- **CommonEdge**: 通用连接线（支持箭头、虚线等样式）
- **MindMapRect**: 思维导图节点矩形
- **MindMapLine**: 思维导图连接线（支持贝塞尔曲线）

#### ShapeManager - 图形管理器
- **功能**: 管理图形组件的注册和获取
- **特性**:
  - 支持动态注册新的图形组件
  - 提供统一的图形组件访问接口
  - 使用 Map 结构存储组件映射关系

### 4. 事件系统

#### 事件处理机制
- **createEventHandler**: 为图形组件创建统一的事件处理器
- **支持事件类型**:
  - 鼠标事件：click, mousedown, mouseup, mouseover, mousemove
  - 拖拽事件：dragover, drop
  - 双击事件：dblclick
- **事件传播**: 使用 `Emitter` 进行事件的发射和监听

#### 事件类型定义
```typescript
- SHAPE_CLICK: 图形点击
- SHAPE_MOUSE_OVER: 鼠标悬停
- SHAPE_MOUSE_DOWN: 鼠标按下
- SHAPE_MOUSE_UP: 鼠标释放
- SHAPE_MOUSE_MOVE: 鼠标移动
- SHAPE_DRAG_DROP: 拖拽释放
- SHAPE_DRAG_OVER: 拖拽悬停
- SHAPE_DBL_CLICK: 双击
```

### 5. 工具类系统

#### PathBuilder - SVG路径构建器
- **功能**: 提供便捷的SVG路径构建方法
- **支持操作**:
  - 移动：moveTo, MoveTo
  - 直线：lineTo, LineTo
  - 水平线：lineHorizontalTo, LineHorizontalTo
  - 垂直线：lineVerticalTo, LineVerticalTo
  - 路径闭合：closePath
  - 贝塞尔曲线等高级路径操作

#### Point - 坐标点工具
- **功能**: 处理二维坐标点的计算和变换
- **应用**: 图形定位、路径计算、碰撞检测等

## 接口定义

### GraphProps - 图形属性接口
```typescript
interface GraphProps {
    edges: Shape[]      // 边线集合
    symbols: Shape[]    // 符号图形集合
    graph: GraphModel   // 图形模型实例
}
```

### IGraphOption - 图形选项接口
```typescript
interface IGraphOption {
    graph: GraphModel
    customEndMove(moveModel: MoveModel, dx: number, dy: number): Promise<any>
    showPopover(index: VertexType, shape: Shape): void

    addMindMapRect(index: VertexType, shape: Shape): void
    saveText(shape: Shape, text: string): Promise<void>
    expandShape(shapeId: string, expand: boolean): Promise<void>
}
```

## 核心功能特性

### 1. 图形渲染
- **SVG 基础**: 使用 SVG 技术实现矢量图形渲染
- **组件化**: 每种图形类型对应独立的 Vue 组件
- **样式支持**: 支持丰富的图形样式（颜色、边框、填充、虚线等）
- **响应式**: 图形属性变化时自动重新渲染

### 2. 交互功能
- **选择操作**: 支持单选、多选、框选
- **拖拽移动**: 图形的拖拽移动和位置更新
- **缩放平移**: 画布的缩放和平移操作
- **标签编辑**: 双击图形进行文本标签编辑
- **悬停效果**: 鼠标悬停时的视觉反馈

### 3. 思维导图支持
- **节点渲染**: 专门的思维导图节点组件
- **连线绘制**: 贝塞尔曲线连接线
- **层级管理**: 支持多层级的思维导图结构
- **展开折叠**: 节点的展开和折叠功能

### 4. 连接线系统
- **路径计算**: 自动计算连接线的最优路径
- **箭头支持**: 支持起始和结束箭头
- **样式定制**: 支持不同的线条样式和颜色
- **路径点编辑**: 支持连接线路径点的编辑

- 新图形组件只需实现标准的 Vue 组件接口
- 支持自定义图形的样式和交互行为

### 2. 事件系统扩展
- 可以通过 `Emitter` 添加新的事件类型
- 支持自定义事件处理逻辑
- 事件处理器支持选择性忽略特定事件

### 3. 模型扩展
- 可以在 `GraphModel` 中添加新的子模型
- 支持自定义状态管理逻辑
- 响应式数据更新机制

## 性能优化

### 1. 渲染优化
- 使用 Vue 3 的响应式系统，只在数据变化时重新渲染
- SVG 矢量图形，支持无损缩放
- 组件级别的更新，避免全局重绘

### 2. 事件优化
- 事件委托机制，减少事件监听器数量
- 支持事件处理器的选择性注册
- 防抖和节流机制（在具体实现中）

### 3. 内存优化
- 使用 `markRaw` 标记组件，避免不必要的响应式包装
- 合理的组件生命周期管理
- 及时清理事件监听器和定时器

## 开发指南

### 1. 添加新图形类型
1. 在 `src/shape/` 目录下创建新的 Vue 组件
2. 实现图形的 SVG 渲染逻辑
3. 在 `src/shape/index.ts` 中注册新图形组件
4. 在类型定义中添加对应的图形类型枚举

### 2. 扩展交互功能
1. 在相应的模型类中添加新的状态属性
2. 在 `createEventHandler` 中添加新的事件处理逻辑
3. 在图形组件中绑定新的事件处理器
4. 更新事件类型定义

### 3. 自定义样式
1. 在图形组件中添加样式计算逻辑
2. 支持从 `shape.style` 中读取自定义样式属性
3. 在 SVG 元素上应用样式属性
4. 确保样式的响应式更新

## 重要注意事项

1. **坐标系统**: 使用 SVG 坐标系统，注意坐标变换和缩放的处理
2. **事件冒泡**: 注意 SVG 事件的冒泡机制，适当使用 `stop` 修饰符
3. **性能考虑**: 大量图形时注意渲染性能，考虑虚拟化或分层渲染
4. **类型安全**: 充分利用 TypeScript 的类型检查，确保代码的健壮性
5. **组件通信**: 使用事件系统进行组件间通信，避免直接的组件依赖

## 总结

`@hfdraw/graph` 是一个功能完整、架构清晰的图形渲染引擎。它采用组件化设计，支持多种图形类型，提供丰富的交互功能，具有良好的扩展性和性能表现。该包为 HfDraw 项目提供了强大的图形绘制和编辑能力，是整个绘图应用的核心基础。
