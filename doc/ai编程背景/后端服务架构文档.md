# 后端服务架构文档

## 项目概述

`@hfdraw/services` 是一个基于 NestJS 框架的后端服务，为绘图应用提供数据持久化、实时通信和业务逻辑处理。该服务采用模块化架构设计，支持多项目管理、图形数据操作、步骤管理和 WebSocket 实时通信。

## 技术栈

### 核心框架
- **NestJS**: 企业级 Node.js 框架，提供依赖注入、模块化架构
- **TypeORM**: ORM 框架，用于数据库操作和实体管理
- **Better SQLite3**: 轻量级嵌入式数据库，支持高性能读写
- **WebSocket (ws)**: 实时双向通信协议

### 开发工具
- **TypeScript**: 类型安全的 JavaScript 超集
- **Class Validator**: 数据传输对象验证
- **JSZip**: 项目文件压缩和解压
- **AsyncLock**: 异步锁机制，确保数据一致性

## 核心架构

### 1. 主要模块

#### ShapeModule (图形管理模块)
- **ShapeController**: 处理图形相关的 HTTP 请求
- **ShapeService**: 图形业务逻辑处理
- **功能特性**:
  - 侧边栏拖拽创建图形
  - 图形移动和变换
  - 图形样式更新
  - 思维导图节点管理
  - 连接线管理

#### ProjectModule (项目管理模块)
- **ProjectController**: 项目相关 API 接口
- **ProjectService**: 项目业务逻辑
- **功能特性**:
  - 项目创建和删除
  - 项目保存和加载
  - 项目列表管理
  - 项目数据库同步

#### StepModule (步骤管理模块)
- **StepService**: 操作步骤管理
- **功能特性**:
  - 撤销/重做操作
  - 步骤历史记录
  - 变更追踪

#### SocketModule (WebSocket 通信模块)
- **WsService**: WebSocket 服务管理
- **WsClient**: 客户端连接管理
- **功能特性**:
  - 实时数据同步
  - 多客户端协作
  - 项目订阅机制

### 2. 数据模型

#### 核心实体

**ShapeEntity (图形实体)**
```typescript
{
  id: string,              // 图形唯一标识
  projectId: string,       // 所属项目ID
  shapeType: string,       // 图形类型
```

### 3. 数据库架构

#### 双数据库设计
- **系统数据库**: 存储应用级数据（项目列表、用户信息等）
- **项目数据库**: 每个项目独立的数据库文件，存储项目相关数据

#### 连接管理策略
- **写连接**: 单一连接，串行写入，确保数据一致性
- **读连接**: 多连接池，并行读取，提高性能
- **连接复用**: 智能连接管理，避免频繁创建销毁

### 4. 事务管理系统

#### StepManager (步骤管理器)
- 统一管理数据库操作
- 自动生成操作步骤
- 支持撤销/重做功能
- 实时变更通知

#### Transaction (事务处理)
- 自动事务管理
- 异常回滚机制
- 连接池管理
- 锁机制保证并发安全

### 5. 实时通信系统

#### WebSocket 架构
- **连接管理**: 自动连接建立和断开处理
- **订阅机制**: 基于项目ID的消息订阅
- **消息分发**: 智能消息路由和广播
- **错误处理**: 连接异常自动恢复

## API 接口设计

### 图形管理 API

```typescript
// 侧边栏拖拽创建图形
POST /shape/sidebarDrop
Body: SideBarDropDto {
  projectId: string,
  point: Point,
  modelId: StType
}

// 获取所有图形
GET /shape/diagram/allShape
Query: FetchAllShapeDto {
  projectId: string
}

// 移动图形
POST /shape/move
Body: MoveShapeDto {
  projectId: string,
  shapeIds: string[],
  dx: number,
  dy: number
}
```

### 项目管理 API

```typescript
// 创建项目

// 获取项目列表
GET /project/list

// 保存项目
POST /project/save
Body: { projectId: string }

// 删除项目
DELETE /project/:projectId
```


## 开发指南

### 1. 新增业务模块
1. 在 `src/modules/` 下创建模块目录
2. 实现 Controller、Service、Module
3. 在 `app.module.ts` 中注册模块
4. 添加相应的 DTO 和实体定义

### 2. 数据库操作
1. 使用 StepManager 进行数据库操作
2. 通过 transaction 函数包装业务逻辑
3. 确保所有写操作都在事务中执行

### 3. WebSocket 消息
1. 使用 WsService 发送实时消息
2. 遵循消息类型定义规范
3. 实现客户端订阅机制

### 4. 错误处理
1. 使用统一的异常处理机制
2. 提供详细的错误信息
3. 实现优雅的错误恢复

## 重要注意事项

### 1. 数据库操作
- 所有写操作必须通过 StepManager 执行
- 避免直接使用 Repository 进行写操作
- 确保事务的正确使用和异常处理

### 2. 并发控制
- 写操作必须获取项目锁
- 避免长时间占用数据库连接
- 合理使用连接池资源

### 3. 性能考虑
- 批量操作优于单个操作
- 合理使用数据库索引
- 避免 N+1 查询问题

### 4. 安全性
- 验证所有输入参数
- 实现适当的权限控制
- 防止 SQL 注入攻击

### 5. 可维护性
- 保持代码的模块化和可测试性
- 编写完整的单元测试
- 维护清晰的 API 文档
