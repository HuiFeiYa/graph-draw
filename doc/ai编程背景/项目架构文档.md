# HfDraw 客户端项目架构文档

## 项目概述

HfDraw 是一个基于 Vue 3 + Electron 的桌面绘图应用，支持流程图、思维导图等多种图形绘制功能。项目采用现代化的前端技术栈，提供丰富的绘图工具和实时协作能力。

## 技术栈

### 核心框架
- **Vue 3** - 前端框架，使用 Composition API
- **Electron** - 桌面应用框架
- **TypeScript** - 类型安全的 JavaScript 超集
- **Vite** - 构建工具和开发服务器

### UI 和样式
- **Element Plus** - UI 组件库
- **SCSS** - CSS 预处理器
- **SVG** - 矢量图形绘制

### 状态管理和路由
- **Pinia** - 状态管理库
- **Vue Router** - 路由管理

### 通信和数据
- **WebSocket** - 实时通信
- **HTTP Client** - API 请求
- **Better SQLite3** - 本地数据库

## 核心功能模块

### 1. 项目管理
- **项目列表** (`ProjectList.vue`): 显示所有项目，支持创建、打开、删除
- **项目创建** (`CreateProject.vue`): 创建新的绘图项目
- **项目状态管理** (`stores/project.ts`): 管理当前项目ID和项目列表

### 2. 绘图编辑器
- **流程图编辑器** (`Flow.vue`): 支持流程图绘制和编辑
- **思维导图编辑器** (`MindMap.vue`): 支持思维导图创建
- **图形编辑器核心** (`editor/graphEditor.ts`): 图形操作的核心逻辑
- **侧边栏组件** (`SiderBar.vue`): 提供图形工具和模板

### 3. 实时协作
- **WebSocket 服务** (`socket/SocketService.ts`): 处理实时数据同步
- **事件总线** (`util/Emitter.ts`): 组件间通信
- **步骤管理**: 支持撤销/重做操作

### 4. 数据服务
- **模型服务** (`util/ModelService.ts`): 项目数据的 CRUD 操作
- **图形服务** (`util/ShapeService.ts`): 图形元素的操作
- **HTTP 客户端** (`util/httpClient.ts`): 与后端 API 通信

## API 接口

### 项目相关
- `GET mainProject` - 获取应用项目
- `POST mainProject` - 创建应用项目
- `POST project/create` - 创建项目
- `POST project/save` - 保存项目
- `POST project/open` - 打开项目
- `GET project/projectList` - 获取项目列表
- `DELETE project/delete` - 删除项目

### 图形操作
- `POST shape/sidebarDrop` - 侧边栏拖拽创建图形
- `GET shape/diagram/allShape` - 获取所有图形
- `POST shape/move` - 移动图形
- `POST shape/updateShapeStyle` - 更新图形样式
- `POST shape/saveText` - 保存文本
- `POST shape/clear` - 清空画布

### 步骤控制
- `POST step/redo` - 重做操作
- `POST step/undo` - 撤销操作
- `GET step/stepStatus` - 获取步骤状态

## 开发指南

### 启动开发环境
```bash
# 安装依赖
pnpm install

# 启动开发服务器
pnpm run dev

# 构建应用
pnpm run build:all
```

### 添加新功能

1. **新增页面**:
   - 在 `src/views/` 创建 Vue 组件
   - 在 `src/router/index.ts` 添加路由配置

2. **新增API**:
   - 在 `src/constants/api.ts` 添加API常量
   - 在相应的Service类中添加方法

3. **新增图形类型**:
   - 在 `@hfdraw/types` 包中定义类型
   - 在编辑器中添加相应的处理逻辑

### 调试技巧

1. **Electron 调试**:
   ```bash
   # 启用调试模式
   pnpm run dev
   # 在 Chrome 中打开 chrome://inspect
   ```

2. **WebSocket 调试**:
   - 在浏览器开发者工具的 Network 标签查看 WS 连接
   - 在 `SocketService.ts` 中添加日志

3. **状态调试**:
   - 使用 Vue DevTools 查看 Pinia 状态
   - 在组件中使用 `console.log` 输出状态变化

## 架构特点

### 1. 模块化设计
- 编辑器核心与UI分离
- 服务层统一管理API调用
- 组件职责单一，易于维护

### 2. 实时协作
- WebSocket 实现实时数据同步
- 事件驱动的架构支持多用户协作
- 操作历史记录支持撤销/重做

### 3. 跨平台支持
- Electron 提供桌面应用能力
- Web 技术栈保证跨平台兼容性
- 本地数据库存储保证离线可用

### 4. 可扩展性
- 插件化的图形编辑器
- 模块化的包管理 (monorepo)
- 类型安全的 TypeScript 支持

## 注意事项

1. **性能优化**:
   - 大量图形元素时注意虚拟化渲染
   - WebSocket 消息频率控制
   - 图形操作的防抖处理

2. **数据安全**:
   - 本地数据库文件的备份机制
   - 网络传输的数据加密
   - 用户操作的权限控制

3. **兼容性**:
   - Electron 版本与 Node.js 版本匹配
   - 不同操作系统的路径处理
   - 浏览器兼容性考虑
